《谁是AI》—— AI 玩家系统

一款社交聊天推理游戏：真人与隐藏的 AI 混在一起，靠聊天与盲投找出 AI。
这套系统要把 AI 玩家做到真人难以分辨，并用一套离线机制持续把它变更强。

L0 · 一图看懂

在线 ⇄ 离线，一个互喂的闭环

整套系统交付两样东西，并让它们持续互相喂养。滚动跟随高亮，看「它们怎么串起来」。

交付两样东西

A · 一个真人难以分辨的在线 AI 玩家——它在真实对局里聊天与投票。
B · 一套能持续把 A 变更强的离线优化机制——自动迭代它的提示词。

在线层 · 生产

AI 玩家就是当前的 champion 提示词 + 一张人设卡 + 投票逻辑。它在真实房间里和真人聊天、盲投，唯一目标是别被识破、撑过 4 轮。

champion 上线 ↑

离线沙盒每迭代出更强的一版提示词，就上线成为新的在线 AI 玩家。这是离线对在线的喂养方向。

离线层 · 优化沙盒

用 LLM 自对弈：对局引擎 → 裁判评分 → 评分聚合 → 优化器 → 编排器，自动改提示词，代代复利地变强。

真值 + 失败回灌 ↓

在线真人对局产出两种养料：真值（校准沙盒的代理指标）和真实失败（回灌沙盒的场景库）。这是在线对离线的喂养方向。

闭环 · 棘轮

真人对局永远是最终裁决，沙盒只是放大真人信号、快速预筛的离线加速器。两条喂养方向合起来，就是一个只进不退的优化闭环。

L1 · 三层架构

三层 + 连接它们的契约

把闭环拆开看：每一层各司其职，靠稳定的 schema 解耦，可独立替换。

在线层

AI 玩家提示词 + 人设卡 + 投票逻辑。这是被优化的对象，也是唯一真正在生产环境里玩的东西。

离线优化层

六个模块组成自对弈流水线：对局引擎 · 裁判评分 · ★评分聚合 · 优化器 · 编排器 · 场景库，把 AI 玩家迭代出更强的 champion。

契约层

拆成两份单一事实来源：输入契约（场景/探测的 schema）与输出契约（对局记录的 schema）。让模块只对接口编程、可独立替换，防字段被单方需求带偏。

参与方

侦探 / 填充 / 裁判三套提示词在沙盒里扮演对手与评委，构成一个可控、像真人的对局环境——既要够像真人，又要守住「反指纹」纪律不失真。

L2 · 离线流水线

一局对局的旅程

跟着一局对局从场景库出发，看它如何一站站被加工成「让 AI 变强」的信号。滚动驱动小车前进。

站 0 · 场景库

不穷举：状态空间无穷，穷举不可达且仍会漏真人的骚操作。改用七维分层抽样，主要靠真人失败回灌生长，分 optimize / holdout 两 split。

产物 → Scenario（输入契约）

站 1 · 对局引擎

按场景把一局从头跑到尾，只跑不评分。关键：scripted_intent/free 两模式、full_match/spotlight 两形态、探测 = 触发器 + 意图、vote_policy 三档（默认 live）、同种子配对降方差。

产物 → MatchRecord（可复现：转录 + 投票 + probe_events）

站 2 · 裁判评分

把一局变成 ScoreRecord。两类信号：客观结果指标（真值）+ 裁判 LLM 两遍法——① 匿名盲测可疑度（像玩家不像法医）② 已知 AI 的八维诊断 + 失败定位。

产物 → ScoreRecord

站 3 ★ · 评分聚合（命门）

整套沙盒最核心的一环。成百上千条单局分要被压成几个明确信号。四层严格自底向上：
run 取均 → (scenario,seed) 配对做差 → scenario 收敛(N=场景数) → bootstrap 估 CI → 卡 verdict。

纲领：平均压随机噪声，做差消共模干扰。 聚错一步（尤其 N 与顺序），整套迭代就很努力地朝噪声跑。

产物 → 验证信号 / 瞄准信号 / 失败案例

站 4 · 优化器

进化的智能变异算子：吃瞄准信号 + 失败案例 + 失败记忆，单候选产 1 个带可证伪假设的子版本；K 个子版本靠外层分配 K 个靶子（破绽 × 算子）跑 K 次。原则：一次一破绽、小步不重写、降被识破不刷分。

产物 → k 个子版本提示词

站 5 · 编排器

总控与三道闸的执行处。代际循环：采样父代 → 外层派靶 → 优化器提案 → 配对评测（子 vs 父，同种子）→ 接受闸门（可疑度显著降 + 不否决 + 不回退）→ 留出复核 → 晋升 champion / 进种群 → 周期真人校准 + 回灌。

产物 → 新 champion → 回到在线层上线

贯穿全局 · 防自欺

三道闸门：只进不退的棘轮

为什么这套环不会「越迭代越自欺」？因为每一代都必须同时通过三道闸。噪声驱动的假进步过不了重复确认，只有真实增益代代复利。

每过一道闸 = 棘轮转一齿，退不回去

闸 1

配对 + 显著性

防 · 噪声假信号

子 / 父同场景同种子，只让 AI 提示词当变量；同种子做差消掉共模噪声；统计样本单元用场景数不是对局数；差异超噪声带且够大（MDE）才算赢，否则保留旧版。

闸 2

留出集 + 探测轮换

防 · 过拟合（背题）

验收用的场景与探测实例永不进优化器；偏好提示词的通用回避反射而非死记答案；盯住 recurring——库里有还犯 = 在背答案。验的是「会回避」，不是「背题」。

闸 3

真人校准

防 · Goodhart（优化代理而非真人）

每 M 代用真人对局做相关性体检：沙盒分涨，真人是否也更难识破？真人永远是最终裁决，沙盒只是快预筛器；一旦翻车立即回滚。
