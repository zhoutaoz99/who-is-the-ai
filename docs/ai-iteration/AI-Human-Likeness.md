## 1. 背景

这一轮优化的起点很简单：AI 能说话，但说得像系统在复盘，而不像群里的一个玩家。

最早的问题不在“模型不会写中文”，而在于：

- 低信息时也要硬说，像在完成任务。
- 回复过完整，像总结而不是临场聊天。
- 多个 AI 的语气和节奏太像，像同一套模板。
- 投票和发言之间缺少自己的历史痕迹，不够自洽。

因此，这篇文档记录的是“怎么把 AI 从分析器推回玩家”的过程。

## 2. 问题陈述

### 2.1 低信息场景仍强行输出

原始发言经常把一句很轻的寒暄，扩展成完整分析。

```text
3号好，刚打了个招呼。2号怎么不说话呀？先看看大家反应。
```

问题不在于内容错，而在于说法太像系统总结。真人在这种场景里更可能只丢一句短话。

更自然的方向是：

```text
就一个 hi 也看不出啥，2号人呢？
```

### 2.2 多人同时点评，过于均衡

原始回复常把几个人都照顾到，听起来像在做会议纪要。

```text
哈哈，1号别急，我刚在看大家发言呢。3号反应挺快的，1号也挺主动的。先听听看。
```

问题在于过于均衡，缺少临场站位。真人通常只盯一个点，不会一口气把所有人都评一遍。

### 2.3 被质疑时回复太完整

被点名后，模型容易给出一整套防守链路。

```text
3号，你第一反应说我机械，是不是有点敏感了啊？这才刚开局呢，我让大家说话不是很正常吗？你反应也太快了点吧。
```

方向没错，但太完整、太工整，像标准答辩。真人更常见的是短反问、局部解释、带一点情绪。

## 3. 假设

这一轮我们验证了几个判断：

- 抽象策略字段会把模型推回模板话术。
- 没有短期记忆，发言和投票很容易互相打架。
- 所有人共用一套语气规则，会让不同 AI 收敛成同一种人。
- 只靠调度，不能解决“说什么像不像真人”的问题。

## 4. 改动项

### 4.1 把策略层从抽象词改成具体动作

第一步是把旧的“goal / reason / intensity / length”换成更具体的结构化策略：

```json
{
  "replyTo": "3号说我机械",
  "speechAct": "防守反问",
  "publicPoint": "我只是催2号说句话，不足以说明机械",
  "tone": "有点不服，但别长篇解释",
  "maxSentences": 2,
  "constraints": ["不要同时点评多人"],
  "avoidPhrases": ["先看看大家反应", "带节奏", "有点可疑"]
}
```

这个改动的目标很明确：让模型先决定“接哪句话、用什么动作、只说一个什么点”，而不是先把一整段分析写出来。

### 4.2 把表达层压回临场聊天

表达层不再负责分析，只负责把策略翻译成最终发言。

这一步引入了几条硬约束：

- 默认 1-2 句。
- 最长 240 字符。
- 允许口头语、反问、省略。
- 不允许报告体、主持体、复盘体。
- 不允许直接复述策略字段。

实际效果是，模型从“写完整论证”逐渐变成“只接一个点，然后停下”。

### 4.3 引入投票短期记忆

为了让后续发言和投票不互相打架，我们把自己的投票历史存进了房间短期记忆：

```ts
type AiShortMemory = {
  votes: Array<{
    roundNo: number;
    targetSeatNo: number;
    publicReason?: string;
    source: "model" | "fallback";
  }>;
};
```

关键约束是：

- 只记录自己的公开投票。
- 只保留最近 4 条。
- 投票真正写入后才记忆。
- 兜底投票不伪造理由。

这一步把“我刚才投了谁、为什么投”变成可回收上下文，后续说话会更自然。

### 4.4 接入人格库

我们把 AI 的语气差异从单个 prompt 文案里拆出来，放进人格库。

人格不再只是“一个风格说明”，而是一个可以被版本化和热切换的集合。当前库覆盖的方向包括：

- 热络破冰
- 划水摸鱼
- 贫嘴玩笑
- 暴躁直球
- 表情语气
- 社恐慢热
- 认真分析
- 杠精抬杠

这些人格通过 `speechStyle`、`sentenceStyle`、`responseBias`、`toneRules`、`avoidPhrases`、`typingHabit` 等字段进入 prompt。

### 4.5 让调度从固定轮询变成观察驱动

单靠固定频率轮询，会让 AI 看起来像定时器在说话。

所以我们把发言流程改成：

- 先观察局势。
- 决定现在说不说。
- 如果说，给出目标反应时间。
- 如果不说，给出下一次观察时间。

这一步并不是在“让 AI 更能聊”，而是在让它更像一个会等会看会插话的玩家。

1. 回放样本：

观察维度主要看：

- 开场句是否还在复用模板。
- 被质疑后是否还是长篇答辩。
- 第一轮是否还会过早催票。
- 投票和发言的自洽性是否提升。
- 不同人格之间的语气差异是否明显。

## 6. 结果

这几轮改动后，能确认的变化有：

- 低信息场景下，AI 更常 skip 或只说一句短话。
- 发言长度明显下降，模板味减少。
- 被点名时更容易短反问，而不是完整防守。
- 投票记录开始能回流到后续发言里。
- 同一套策略在不同人格下，口吻差异更明显。

同时也看到了新的瓶颈：

- 真正难的已经不是“会不会说话”，而是“该不该说、该投谁、怎么和阵营目标对齐”。
- 策略层一旦过抽象，表达层还是会回到模板。
- 人格如果写得太强，容易压过内容判断。

## 7. 结论

这轮迭代的结论是：AI 拟人化不是一个 prompt 文案问题，而是一个组合问题。

要让 AI 看起来像人，至少要同时处理四件事：

1. 说不说、什么时候说
2. 说什么动作
3. 用什么口吻说
4. 之前自己说过什么、投过什么

也就是说，真正有效的改法不是继续堆示例，而是把策略层、表达层、短期记忆和人格库拆开，各自负责一段。

## 8. 下一步

后续如果继续迭代，优先顺序会是：

1. 继续收集低信息局、被质疑局和第一轮局的回放。
2. 扩充易触发模板化的禁用句式。
3. 观察人格库是否需要再细分，避免几个角色过于接近。
4. 如果投票仍然不自洽，再考虑扩展记忆内容，而不是先加更多文案。
