## 1. 背景

提示词优化器（champion 迭代）只优化**一件东西**：`ai_under_test` 的讨论 `system` 提示词正文。

- 讨论 system（仅 `ai_under_test`）：`buildDiscussionSystemPrompt` 走 `loadPromptVersionText(champion) || system-discussion.txt`（`apps/api/src/ai/ai.service.ts:446`）——优化器能改的就是这块 DB 文本。
- 投票 system（`buildVoteSystemPrompt`，`ai.service.ts:458`）、侦探/填充提示词（`renderTemplate`）是**静态文件**，根本不在优化环里。

因此有一整类"AI 破绽（tell）"落在优化器射程之外：它们不是"system 文本写得好不好"的问题，而是**模型输入里缺状态、模型输出的投递方式、发言轮次的仲裁**——这三层都不是一次无状态文本补全能自己控制的，也不是改 system 措辞能补的。

本文把这类缺口登记成册，作为**问题登记表（register）**：确认现状（事实）、说明为何优化器不可解、给出候选机制（待决）、标注与"健忘/不一致"设计取向的张力，供后续单独立项。

> **当前快照（2026-07-02）**：按[系统级健康度关注清单](../../audit/System-Health-Watchlist.md)，自进化闭环已建好但从未真跑过一代，champion 仍是种子。但本文结论与闭环是否在跑**无关**——即便优化器满负荷运转，下列缺口仍在它射程外。

## 2. 目标

- 给出"某问题属于**优化器不可解的系统级缺口**"的**判定标准**（§3），让新问题能被一致地分类。
- 登记已识别的缺口，每条包含：现状事实、为何不可解、候选机制、风险张力、度量方式。
- 明确**建议不做**的候选及理由，避免反复讨论。

## 3. 非目标

- **不**为任何单个缺口给出定稿设计。每条都是候选机制、状态 `待决`；立项时应另起 Design/ADR。
- **不**讨论 system 提示词措辞如何优化——那是优化器/champion 的职责，见[提示词与人设卡](./ai玩家%20prompt模板与人设卡.md)。
- **不**替代[真人校准](../离线优化沙盒/模块详细设计/5.真人校准%20·%20方案设计.md)（离线评测保真度）或[健康度清单](../../audit/System-Health-Watchlist.md)（闭环监督）。

## 4. 判定标准：什么是"优化器不可解"

一个拟人化问题属于本文范畴，当且仅当它命中以下**任一类**（优化器只能改 system 文本，这三类都改不动）：

| 类别 | 定义 | 为什么改 system 文本没用 |
| --- | --- | --- |
| A · 输入状态缺失 | 模型输入里没有这份状态，且数据未被采集/回注 | 你没法用文字"告诉"模型一份它看不到的数据 |
| B · 输出投递方式 | 问题在输出的时序、分段、打字噪声上 | system 只影响生成内容，不控制"怎么发出去" |
| C · 轮次仲裁 | 问题在"谁该此刻发言"的调度，发生在调模型**之前** | 模型无法给自己排轮次；调度在它被调用前已决定 |

反例（**不**属于本文，归优化器）：打法过于统一、回避过度、点名 vs 低存在感的矛盾——这些是 system 文本能调的度，应交给优化器搜索。

## 6. 详细缺口

每条按：**现状（事实）→ 真人对应能力/为何是 tell → 为何优化器不可解 → 候选机制（待决）→ 风险与张力 → 度量**。

### 6.1 `GAP-TIMING` 时间感知与节奏

- **现状（事实）**：注入模型的聊天记录是 `label: content`，**无时间戳/间隔**（`formatConversation`，`ai.service.ts:911`）；`message.createdAt` 已存储但未渲染。`GameContext` 里有 `remainingTimeMs` 等字段但两个 user 模板都没注入。出站延迟 `typingDelayForContent`（`ai.service.ts:906`）只按**字数**估算。
- **为何是 tell**：真人能感知消息间隔、房间语速、自己是否"秒回"，并据此压哨补话、错峰发言。模型对时间零感知，回复节奏由外部长度公式决定，规律性本身就像机器。
- **为何优化器不可解**：命中类别 A + B。信号不在输入里，投递时序也不由 system 决定；改措辞给不了模型一块表。
- **候选机制（待决）**：① 转录注入相对时间/间隔；② 节奏控制器按"消息类型 + 房间当前语速"从人类分布采样延迟（替代纯长度公式）；③ 注入"本阶段剩余时间"让 AI 会压哨。
- **风险与张力**：低，基本纯增益。过度规律的"拟人抖动"若又形成新规律仍是 tell，需带随机性并校准。
- **度量**：AI 与真人的"消息间隔分布"KS 距离；盲测可疑度中"回复太快/太规律"类失分。
- **实现进度**：候选机制 ③ 的**输入侧第一步已落地（2026-07-02）**——讨论提示词注入与真人同源的粗粒度剩余时间，记录见 [改动记录 · 2026-07-02](./AI拟人化系统级缺口-改动记录.md)；候选机制 ①②（相对时间戳、输出侧节奏控制）仍未做，本缺口整体未完结。

### 6.2 `GAP-SEGMENTATION` 发言分段与打字噪声

- **现状（事实）**：`generateSpeech` 返回**单条** `content`（`AiSpeechAction`，`ai.service.ts:317`）；无分段、无打字噪声后处理，`cleanSpeech` 只做清洗不做拟人扰动。
- **为何是 tell**：真人常是多气泡爆发（"等下"…"我觉得5号怪"…"*怪 打错了"），有错字、掉标点、事后更正；模型输出一条干净完整句。system 里那句"刻意装人也会露馅"已承认：让模型**自己伪造**错字/分段只会漫画式失真。
- **为何优化器不可解**：命中类别 B，且是模型风格生成的固有弱项。
- **候选机制（待决）**：确定性后处理 humanizer——按概率把一条回复拆成 1~3 气泡（配合 `GAP-TIMING` 的气泡间延迟）、按可调噪声率注入错字/大小写/标点扰动、偶发更正气泡。基线取自真人校准数据。
- **风险与张力**：低~中。过度必然更假，须可开关、按人类基线校准并度量。
- **度量**：每条发言气泡数、错字率、句长分布与真人基线的差异；盲测中"过于工整"类失分。

### 6.3 `GAP-SELF-CONSISTENCY` 自洽事实账本

- **现状（事实）**：转录含 AI 过去发言，但随记录变长早期自述显著性下降；无"AI 已立过的硬事实"紧凑状态。
- **为何是 tell**：真人记得自己背景设定（第 1 轮说"我上海的"，第 3 轮不翻供）；模型可能与自己的人设事实打架。
- **为何优化器不可解**：命中类别 A——是"抽取 + 持久化 + 回注"的机制，不是措辞。与 `GAP-VOTE-MEMORY` 同族。
- **候选机制（待决）**：每轮抽取 AI 关于自己的**硬事实**断言（城市/年龄/职业等），存储并紧凑回注。
- **风险与张力**：中。设计明确要"允许自相矛盾、别严丝合缝"（见 v4.0 拟人化设计）。故账本**只锁硬事实，观点/情绪的软漂移必须放开**，否则把 AI 推回机器味。
- **度量**：跨轮硬事实自相矛盾出现率；盲测中"前后不一致"被点名率。

### 6.4 `GAP-ADDRESSIVITY` 被点名/围攻的轮次仲裁

- **现状（事实）**：`selectSpeechPlayer`（`game.service.ts:2023`）按冷却 + `aiSkipBackoff` + `selectByRoundFreshness` 挑人，**不看"谁刚被直接点名/提问"**。
- **为何是 tell**：被 @ 追问的 AI 可能轮不到发言（对着直球装死，可疑），或没被问的 AI 突兀插话；真人是"被点就答、没被点就让"，被指认时其他人会快速跟投围攻。
- **为何优化器不可解**：命中类别 C——发言轮次在**调模型之前**由调度器决定，模型无法给自己排序；"我正被针对"也是需计算的状态。
- **候选机制（待决）**：调度器 addressivity-aware（被点名席位提权、有人被指认时其他人快速跟投）；可把"你刚被直接问了 X / 本轮 N 人指向你"作为信号注入。
- **风险与张力**：低~中。已有调度器骨架可扩展。
- **度量**：AI 被直接点名后的应答延迟/应答率与真人对比；"装死不应答"被识破率。

### 6.5 `GAP-IMPERFECT-ATTENTION` 不完美注意力

- **现状（事实）**：模型完整、精准吞下整段转录，具备超人回忆——能准确引用三轮前的细节。
- **为何是 tell**：走神的真人早忘了；完美、完整的复盘本身就是 AI 签名。
- **为何优化器不可解**：命中类别 A 的**反向**——只能从输入里**扣掉**数据（加窗、随机省略若干行、"这几条你没看到"），无法靠提示词可靠地让模型"忘记/略读"。
- **候选机制（待决）**：受控地降级注入转录（滑动窗口/随机丢行/标注"你走开了这几条"）。
- **风险与张力**：会牺牲一点决策质量——但那恰恰更像人。优先级低，先观察是否真构成可测 tell。
- **度量**：AI 引用"久远细节"的精确度与真人对比。

## 7. 起点实例：`GAP-VOTE-MEMORY`（已讨论，结论：不回注）

- **现状（事实）**：`AiShortMemory`（`game.types.ts:88`）+ `rememberAiVote`（`game.service.ts:2529`）**已在采集**每轮投票的"投给谁 + 理由 + 来源 + 策略"，保留最近 4 轮。`buildGameContext` 把它塞进 `context.shortMemory`（`game.service.ts:2523`），但**没有任何提示词模板引用它**。其唯一生效消费方是审计：`match-record/build.ts:47` 把 `reason`/`policy_applied` 挂到 MatchRecord。
- **为何登记在此**：它是"输入状态缺失"类别 A 的典型——数据在，但没回注给模型；这正是本文其余缺口的原型。
- **结论（判断）**：**不建议把私有投票理由回注提示词**。① 提示词里已注入**公开** `voteHistory`（谁投谁、票型、出局），投票连续性已足够；② 回注"你上几轮投谁、理由是 X，保持一致"会把 AI 推向"记得住、前后一致、会复盘"——正好违反 v4.0 的"健忘/凭感觉/不复盘"取向，增加暴露风险。除非出现"每轮改投 + 理由互相打脸被真人一眼看穿"的具体 badcase，否则维持现状。
- **收尾项**：`context.shortMemory` 是接好但空跑的字段，留着无害；要清爽可删除其在 context 的传递（审计走 `room.aiMemories`，不受影响）。

## 8. 明确不做：`GAP-CROSS-ROUND-PLAN` 跨轮策略连续性

"每次调用无状态重推策略、无携带计划"看似缺口，但补上持久化意图会让 AI **更有目的、更前后一致**，直接撞上"别太一致、会健忘"的设计取向与人类相似度目标，多半**负收益**。除非观测到具体的"策略精神分裂"badcase，否则不做。

## 9. 与优化器 / 真人校准 / 沙盒的关系

- **优化器**：只改讨论 system 文本，够不到本文缺口（§1、§4）。本文缺口应作为**独立机制**立项，不占用优化器搜索空间。
- **真人校准**（[方案](../离线优化沙盒/模块详细设计/5.真人校准%20·%20方案设计.md)）：离线评测保真度机制，不改运行时行为；但其采集的真人日志正好可为 `GAP-TIMING`/`GAP-SEGMENTATION` 提供**分布基线**。
- **沙盒盲测可疑度**：可作为这些机制"是否降低被识破率"的离线代理指标（注意其本身也是代理，最终以真人识破率校验）。

## 10. 风险与失败模式

| 风险 | 触发条件 | 影响 | 缓解措施 |
| --- | --- | --- | --- |
| 拟人化过度 | humanizer/节奏抖动参数过猛 | 比"太工整"更假，反向暴露 | 参数可调 + 按真人基线校准 + A/B 度量 |
| 新规律形成 | 时序/分段用固定规则 | 规律性本身成新 tell | 引入随机性，分布匹配而非定值 |
| 一致性过头 | 自洽账本锁到观点层 | AI 变得机器味 | 只锁硬事实，放开软漂移 |
| 决策质量下降 | 不完美注意力删太多上下文 | 明显失智被识破 | 小幅、受控降级并度量 |
| 与优化器耦合 | 把这些改成 system 文本硬编码 | 被 champion 覆盖 / 缩小搜索空间 | 落到运行时机制层，不进 system 文本 |

## 11. 验证方式

- **离线**：在沙盒对同一 champion 做开/关机制的对照，看盲测可疑度、AI 被票出率是否下降。
- **分布对齐**：对 `GAP-TIMING`/`GAP-SEGMENTATION`，用真人校准数据比对消息间隔/气泡数/句长分布（KS 距离等）。
- **真值**：最终以真人对局的 AI 识破率校验（真人校准主线），防止只骗过 LLM 代理。

## 12. 后续工作

- 优先立项 `GAP-TIMING` 与 `GAP-SEGMENTATION`（Tier 1，纯机制、现实中最硬的 tell，已有 `typingDelayForContent` 与出站延迟通道可挂），各出独立 Design/ADR。其中 `GAP-TIMING` 输入侧已落地（见 §6.1 实现进度），剩输出侧节奏待做。
- 次之 `GAP-ADDRESSIVITY`（调度器已有骨架）、`GAP-SELF-CONSISTENCY`（注意只锁硬事实）。
- `GAP-IMPERFECT-ATTENTION` 先观察是否构成可测 tell 再决定。
- 新识别的拟人化问题先用 §4 判定标准归类：属优化器射程的交优化器，命中 A/B/C 的登记进本表。
