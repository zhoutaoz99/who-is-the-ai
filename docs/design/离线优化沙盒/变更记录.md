## #1 移除 `intent_schedule` 与 `base_intent`(2026-06-25)

### 背景
早期设计(见《对局引擎·方案设计》§5.1「固定剧本 + 活反应」)把对手行为拆成「固定外部冲击」与「放活反应」两类,并用 `intent_schedule`(逐轮给对手注入"本轮意图",如"这轮重点怀疑 B")与 `RosterSlot.base_intent`(静态立场补充)来实现"固定"那一半。MVP 据此实现了从契约、运行时到提示词的完整链路。

### 触发
实现收尾时复盘发现:`intent_schedule` 与沙盒最核心的原则存在真实矛盾——它不是"口味选择",而是把对手**往"写死"那个方向**推。

### 依据原则
> **不冻结环境,用统计降方差**:不靠把对手写死来消方差(那会失真),而靠**同种子 + 配对 + 多 run + 加样本**。(见《离线沙盒 模块设计清单》三大设计原则之二、避坑清单「投票别写死」「探测别用死台词」。)

### 决策
**移除** `intent_schedule` 与 `base_intent`(契约字段、运行时 plumbing、提示词注入位、示例场景、前端展示全部清掉)。侦探/填充的立场与打法**完全由其人设卡承载**(静态、固定),逐轮反应交给真实上下文自然涌现。

### 理由:probe / spotlight / intent 的「可固定性」根本不同

| 机制 | 是否外生于 AI 措辞 | 固定它是否扭曲活反应 | 结论 |
|---|---|---|---|
| **probe**(`probe_schedule`) | 是——真人也会不管你说什么、突然甩怪问题 | 否——固定的是"砸什么探测、何时砸",投手当场看真实上下文生成台词 | **保留** |
| **spotlight**(`seed_history`) | 是——把"为何集火/结盟"作为**既成事实**发生在镜头外 | 否——只设定**初始状态**(所有评测都这么干),起跑之后自然演化 | **保留** |
| **intent_schedule** | 否——在镜头内强行规定活侦探"这轮怀疑谁" | **是**——篡改进行中的判断,把活对手写死 | **移除** |
| **base_intent** | 否——可被用来给特定侦探写死"专盯 B"(预 scripting 的滑坡);且人设卡已承载静态立场 | 是 | **移除** |

关键差别:**probe / spotlight 固定的是「外部冲击 / 初始状态」**(合法,公平且不失真),**intent_schedule 固定的是「对手的活反应」**(失真,违背原则)。

### 替代方案(社交局势怎么实现)
"被集火""结盟"等社交局势不再靠注入意图:
- **`full_match`**:让活侦探基于真实上下文自然涌现——AI 表现好则怀疑自然消退,差则加重,这条因果链被保住,正是要测的东西。
- **`spotlight`**:想测"被集火的 R3",就把预置历史写成"前两轮 B 已被几个人盯上",从有来由的处境起跑。集火是**作为历史背景**给定(发生在镜头外),不是镜头内扭转判断。

### 影响范围
- **输入契约**:Scenario 删 `intent_schedule`;RosterSlot 删 `base_intent`;`IntentDirective` 整型移除;校验(§7)删相关不变量。
- **输出契约**:MatchRecord 的 `Turn` 删 `injected_intent`。
- **运行时**:`GameContext` 删 `myBaseIntent`/`myInjectedIntent`;`Room` 删 `sandboxIntentSchedule`;`Player`/`SandboxPlayerSpec` 删 `baseIntent`;`game.service` 删 `resolveInjectedIntent` 及 `createSandboxRoom` 的 intentSchedule 参;`castAiVote` 的 live/rule/scripted 分支不受影响。
- **提示词**:侦探讨论/投票、填充讨论模板删 `{{base_intent}}` / `{{本轮intent}}` 注入位;**保留** `{{本回合任务}}`(probe)。
- **立场来源**:侦探立场完全由人设卡(含 `howToPlay`)承载,逐轮怀疑谁自然涌现。
- **示例场景 / 前端**:5 个示例与配置页清理。
