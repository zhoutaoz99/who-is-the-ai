# 端到端一代闭环 · 流程说明

> **定位**:把"**父版本 → 生成候选子版本 → 配对评测做差 → 选出晋升者**"串成一篇可读的端到端叙事。
> **不重复**各模块内部细节(算法/schema 去看各专文);本文只讲**跨模块的数据流转、时序、两信号区分**,并把这些环节收口到对应专文。
> **读者路径**:新读者先读本文(懂一代闭环怎么转)→ 按需深入《优化器》《评分聚合》《编排器》专文。

---

## 0. 一图速览(一代闭环)

```
┌─ 准备 ───────────────────────────────────────────────────────────────┐
│ 从种群采样父代(champion 必选 + 若干多样高分版);冻结 eval 集            │
│ (optimize 集 + holdout 集,探测实例互不重叠)                          │
└───────────────────────────────┬──────────────────────────────────────┘
                                ▼
┌─ ① 评测父代 ────────────────────────────────────────────────────────┐
│ 父代在 optimize 集上跑 N 场景 × seed × run → 每场一个 ScoreRecord       │
│ → 绝对聚合 + 失败案例聚类 → 【瞄准信号】weak_dimensions + failure_clusters│
└───────────────────────────────┬──────────────────────────────────────┘
                                ▼
┌─ ② 优化器生成候选 ──────────────────────────────────────────────────┐
│ 外层派 K 个靶子(破绽 × 算子;算子战绩 Top-2 做 A/B)                    │
│ 对每个靶子:优化器 LLM 单候选调用(吃瞄准信号+失败案例+失败记忆)         │
│   → 1 个【带可证伪假设】的子版本(完整改后提示词)                       │
│ K 次调用 → validate_prompt 预校验 + 去重 → k 个候选子版本               │
└───────────────────────────────┬──────────────────────────────────────┘
                                ▼
┌─ ③ 配对评测 + 配对做差 ─────────────────────────────────────────────┐
│ 每个候选 vs 父代:用【同一 optimize 集、同一组 seed、同一 run_index】跑   │
│ → 候选 ScoreRecords;buildValidation:                                   │
│   run 取均 → (scenario,seed) 配对做差 → scenario 收敛(N=场景数)         │
│   → bootstrap 估 CI → MDE+CI 卡 verdict                                 │
│ → 【验证信号】每个候选一份(子 vs 父各指标 verdict)                     │
└───────────────────────────────┬──────────────────────────────────────┘
                                ▼
┌─ ④ 接受闸门(优化集)──────────────────────────────────────────────┐
│ 决策只用近真值信号:可疑度【显著】降(≥MDE) + 否决不升                  │
│   + 关键探测不退 + 客观存活不退 → 过闸                                   │
└───────────────────────────────┬──────────────────────────────────────┘
                                ▼
┌─ ⑤ 留出集复核 ──────────────────────────────────────────────────────┐
│ 过闸候选在 holdout 集(用 optimize 从没见过的探测实例)上重验            │
│ 双重意义:验【泛化】(会回避,不是背题)+ 纠【选择偏差】(从 k 个挑赢家)   │
│ 方向一致 + 不显著变差 → 通过                                             │
└───────────────────────────────┬──────────────────────────────────────┘
                                ▼
┌─ ⑥ 落定 ────────────────────────────────────────────────────────────┐
│ 通过 → 晋升 champion / 进种群 top-k;被拒 → tried_and_rejected          │
│ 被接受的假设沉淀为经验;被拒的(含被推翻假设)进失败记忆;更新算子战绩     │
│ → 写 GenerationEval;下一代从更强的 champion 起跑                       │
└───────────────────────────────────────────────────────────────────────┘

跨代:每 M 代真人校准(防 Goodhart)+ 失败回灌入场景库 + 评测集版本化重基线
```

## 1. 两信号:理解整条流程的钥匙

整套闭环里"给优化器/给闸门"的信号是**两条不同的路**,混在一起就会理解错。这是最该先记住的一点。

| 信号 | 回答 | 来源 | 用在 | 形态 |
|---|---|---|---|---|
| **瞄准信号** | champion 现在哪最弱、下一步攻哪? | champion **自己**那批 ScoreRecord(绝对聚合) | ② 生成候选前,喂优化器 | `weak_dimensions`(可靠度加权排序)+ `failure_clusters`(跨局聚类) |
| **验证信号** | 这次改动到底有没有用? | 子 **vs** 父(配对做差) | ④ 接受闸门,判晋升/拒绝 | 每个指标一个 `verdict`(improved/regressed/inconclusive)+ CI |

**正负信号不对称**(关键):

- 优化器**消费瞄准信号 + 失败案例 + 失败记忆**;它**不消费验证信号**——验证信号是它提案**之后**才由评测算出来的产物。
- **正向信号是隐式的**:被接受的改动已经焊进演化后的 champion,优化器下次在更强的基线上继续改。
- **负向信号是显式的**:被拒的改动进 `tried_and_rejected`,显式喂回,防止反复走死路;被推翻的假设也写入,修正优化器对"破绽成因"的因果模型。

> 详见《评分聚合·核心机制》§0(两信号)、《优化器》§1(正负不对称)。

## 2. 角色与记录

**三个 LLM 角色**(彼此独立,降低串谋):
- **AI 玩家**(被测对象):当前 champion 提示词 + 人设 + 投票逻辑,在对局里假扮真人。
- **优化器**(变异算子):吃瞄准信号,产带假设的候选子版本;只提案,不评测不裁决。
- **裁判**:把一局变成 ScoreRecord;产盲测可疑度(决策主代理)+ 客观结果(真值)+ 诊断量表(只诊断,不当优化目标)。

**一代闭环里的关键记录流转**:

| 记录 | 产生方 | 消费方 | 作用 |
|---|---|---|---|
| `MatchRecord` | 对局引擎 | 裁判 | 一局客观发生了什么 |
| `ScoreRecord` | 裁判 | 聚合(→瞄准/验证信号) | 一局的可比分数 |
| 瞄准信号(`weak_dimensions`/`failure_clusters`) | 评分聚合(绝对) | 优化器 | 告诉优化器攻哪 |
| `child`(候选 PromptVersion) | 优化器 | 编排器(配对评测) | 一个带假设的改后提示词 |
| 验证信号(`validation`) | 评分聚合(配对做差) | 接受闸门 | 判这次改动有没有用 |
| `GenerationEval` | 编排器 | 索引库/排行榜 | 一代的接受/拒绝依据 |
| `tried_and_rejected` / `operator_scoreboard` | 编排器维护 | 优化器(下一代) | 失败记忆 / 算子战绩 |

## 3. 一代闭环六阶段(端到端时序)

> 下面每阶段说清三件事:**做什么、产什么、喂给下一阶段什么**。细节回指专文。

### ① 评测父代 → 瞄准信号

- **做什么**:父代(champion)在 **optimize 集**上跑 `N 场景 × seed × run` 真实对局;每局 → MatchRecord → ScoreRecord。
- **产什么**:一批 `championScores`;经**绝对聚合**(非配对)+ 失败案例跨局聚类,得到**瞄准信号**:`weak_dimensions`(可靠度加权排序的弱点)+ `failure_clusters`(高频×高危害×高一致的失败簇,每簇附 1–2 个代表样本)。
- **喂给谁**:② 优化器。
- 注意:这一步**不做配对做差**——配对做差需要"子"的分数,此时还没有。瞄准信号是 champion **自己**的画像。
- 详见《评分聚合》§5.2/§5.3、《裁判》§3/§6。

### ② 优化器生成候选子版本

- **做什么**:
  - **外层派 K 个靶子**:从 `weak_dimensions` 取前几个破绽,主弱点配 2 个算子(按 `operator_scoreboard` 战绩 Top-2 做 A/B)、次要弱点各 1 个,留 1 个自由探索名额(详见《优化器》§2.5/§2.6)。
  - 对**每个靶子**做**一次单候选调用**:把【父代完整提示词 + 指定靶子 + 瞄准信号 + 失败案例 + `tried_and_rejected` + 长度预算 + 锁定段】喂给优化器 LLM → LLM 输出 **1 个带可证伪假设的子版本**(完整改后提示词 + `hypothesis` + `target` + `edit_type`)。
  - **`validate_prompt` 预校验**:锁定段一字不动 / `{{persona}}` 保留 / 长度预算 / 必填齐全 / 不矛盾;**去重**:按 `(target, edit_type)` + 文本相似度。
  - K 次调用聚合 → **k 个过校验的候选子版本**。
- **产什么**:`children[]`(每个是一个 `PromptVersion`,带 `parent_id`/`hypothesis`/`target_dimension`/`edit_type`)。
- **喂给谁**:③ 配对评测。
- 核心纪律:一次一个破绽(可归因)、可证伪假设(闭环验证)、小步不重写、降可疑度不刷分、通用反射不背答案。详见《优化器》§4。

> **子版本怎么"从父版本生成"**:不是代码拼接,是**优化器 LLM 在父版本完整提示词基础上改一处**;代码只负责喂数据 + 校验/格式化 + 组装(`version_id = 父id--算子--随机后缀`,`prompt_text` = LLM 给的整段改后提示词)。

### ③ 配对评测 + 配对做差 → 验证信号

- **做什么**:
  - **配对评测**:每个候选 vs 父代,用**同一 optimize 集、同一组 seed、同一 run_index** 跑对局 → 候选 `ScoreRecords`。
  - **配对做差**(`buildValidation`):严格自底向上——
    `run 取均` → **(scenario, seed) 配对做差** → `scenario 收敛(锁定 N=场景数)` → `bootstrap 估 CI` → `MDE+CI 卡 verdict`。
  - 对每个关心的指标(盲测可疑度 margin / 存活 / plurality / 否决 / 各 probe 通过率)各跑一遍管线。
- **产什么**:**验证信号**——每个候选一份 `validation`(各 form 桶、各指标 `verdict` + `point` + `ci95`)。
- **喂给谁**:④ 接受闸门。
- **为什么配对配在 (scenario, seed)**:场景难度是**共模干扰**(同场景父子同向面对),做差时 `Var(子−父)=Var(子)+Var(父)−2·Cov(子,父)`,高协方差让方差骤降——共模相消,微弱差分信号存活放大。详见《评分聚合》§3.2。

### ④ 接受闸门(优化集)

- **做什么**:决策**只用近真值信号**(可疑度 + 否决 + 客观结果),**绝不用**诊断量表加权总分(易被刷)。过闸需**全部满足**:
  - 盲测可疑度**显著**下降(CI 不含 0 且 ≥ MDE);
  - 否决率不升;关键探测通过率不退;客观存活/plurality 不退。
- **产什么**:每个候选 `promote / reject` 建议 + reasons。
- **喂给谁**:⑤ 留出集复核(仅过闸候选进入)。
- 详见《编排器》§6/§7.1。

### ⑤ 留出集复核

- **做什么**:过闸候选在 **holdout 集**上配对复核,**强制使用 optimize 里从未出现过的探测实例**(措辞隔离)。
- **双重意义**:① 验**泛化**(学的是回避习惯,不是背了 optimize 的题);② 纠**选择偏差**(从 k 个候选里挑出的"赢家"是否真赢,而非配对评测里碰巧幸运)。
- **判据**:方向一致 + 不显著变差即可(holdout 样本通常较少,不强求同等显著)。
- 详见《编排器》§7.2。

### ⑥ 落定:晋升 / 种群 / 失败记忆

- **做什么**:
  - 过留出 → **晋升 champion** / 进**种群 top-k**;被拒 → 写 `tried_and_rejected`。
  - **假设验证回环**:对照候选的 `hypothesis`,目标维度是否真按预测改善——成立则沉淀经验,被推翻则把"成因判断错了"写进失败记忆。
  - **更新 `operator_scoreboard`**:每个被评候选带 `(破绽类型, edit_type, 结果)`,聚合成"哪类改法修哪类破绽最有效"的可量化策略知识。
- **产什么**:`GenerationEval`(一代的接受/拒绝依据)+ 更新后的 champion/种群/失败记忆/战绩。
- **下一代**从更强的 champion 起跑。详见《编排器》§8/§11、《优化器》§7/§11。

## 4. 关键机制(收口指针,不展开)

| 机制 | 在闭环里的位置 | 原理 | 详见 |
|---|---|---|---|
| 父子同 (scenario,seed) 配对 | ③ 配对做差 | 共模相消,差分放大 | 《评分聚合》§3.2 |
| N = 场景数(不是对局数) | ③ scenario 收敛 | 防 CI 虚窄、虚假显著(头号坑) | 《评分聚合》§3.3/§7 |
| 显著性 + MDE | ④ 闸门 | 差异超噪声带且够大才算赢 | 《评分聚合》§3.5/3.6、《编排器》§6 |
| 留出集 + 探测轮换 | ⑤ 复核 | 验泛化 + 纠选择偏差 | 《编排器》§7.2 |
| 算子战绩 + 类型约束 | ② 派靶 | "哪类改法修哪类破绽"随迭代收敛 | 《优化器》§2.6/§7 |
| tried_and_rejected | ⑥ 落定→下一代 | 全局失败记忆,防原地打转 | 《优化器》§7 |
| 种群 top-k + 父代采样 | ① 准备 | 抗局部最优 | 《编排器》§8 |
| 决策 vs 诊断信号分离 | ④ 闸门 | 用近真值信号决策,用丰富信号诊断 | 《裁判》§0、《评分聚合》§0 |

## 5. 防自欺三道闸(在闭环里的落点)

整套闭环能否"稳定演进而不自欺",取决于三道闸都装上:

1. **配对 + 显著性(防噪声)** —— 落在 ③ 配对做差 + ④ 闸门:同种子配做差消场景难度,显著性+MDE 把"手气好"挡在 inconclusive。
2. **留出集 + 探测轮换(防过拟合 + 纠选择偏差)** —— 落在 ⑤ 复核:用没见过的探测验泛化,从 k 个候选里挑赢家时独立重验。
3. **真人校准(防 Goodhart)** —— **跨代**,每 M 代:算"沙盒盲测可疑度 ↔ 真人局结果"相关性,下滑则暂停晋升、修裁判/侦探池;真人为最终裁决,翻车则回滚。

> 详见《编排器》§9、《总体设计》§6。

## 6. 跨代:棘轮与复利

一代闭环不是孤立的——它套在代际循环里,靠**只进不退的棘轮**让真实增益代代复利:

- 即便单代决策偶尔被噪声骗一次,更高层的棘轮(配对+显著 → 留出 → 重基线 → 真人校准)使**假"进步"无法累积**:噪声驱动的假增益过不了重复确认。
- **eval集版本化**(`optimize_v1 → v2`):不同 eval_set 版本的分数**不可直接比**;升级评测集后 champion 重新基线、paired_cache 失效。
- **真人回灌**:真人失败→新场景→回灌场景库→按真人频率重配比;闭环持续逼近真人真实失败分布。
- **free 模式旁路**:周期性用活侦探自由对战跑 champion,发现 scripted 场景里没有的新失败模式,产物喂回灌(不参与一对一判优劣)。

> 详见《总纲》§6/§8、《编排器》§9/§10、《场景库》回灌流程。

---

## 7. 一句话

> 一代闭环 = **父代评测出瞄准信号 → 优化器据此产 K 个带假设的候选 → 每个候选与父代同 (scenario,seed) 配对评测、做差出验证信号 → 优化集闸门 + 留出复核 → 晋升/拒绝并更新失败记忆与战绩**;跨代用真人校准和评测集重基线兜底,只让经得起重复检验的真实增益沉淀下来。瞄准信号告诉优化器"攻哪",验证信号告诉闸门"这次有没有用",两条路不混。
