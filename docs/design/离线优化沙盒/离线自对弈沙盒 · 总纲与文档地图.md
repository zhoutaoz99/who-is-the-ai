# 离线自对弈沙盒 · 总纲与文档地图

> 本文是整套沙盒的**入口与导航**:给出文档地图、全局数据流、贯穿全局的设计原则、各模块摘要+指针、记录索引、端到端循环与避坑清单。
> **各模块的完整细节已拆到对应专文**,本文只在每个模块保留"它是什么 + 关键契约 + 去哪看",不重复 schema 与算法细节。

---

## 0. 这套沙盒在解决什么

目标:在**离线**用 LLM 自对弈,**自动迭代** AI 玩家的提示词,让它在《谁是AI》里更难被真人识破。
做法:跑对局 → 评分 → 聚合成信号 → 优化器改提示词 → 配对评测裁决 → 晋升,代代复利。
成败不在"能不能跑通这个环",而在能否**只让真信号沉淀、不让噪声与自欺累积**——这由三道闸门(见 §3)和评分聚合机制(见 §4 的核心机制专文)保证。

## 2. 全局数据流

```
场景库(分层抽样 + 真人失败回灌)          ── 按[输入契约]产出 ──┐
  ├─ optimize split   ├─ holdout split(含专用探测实例)            │
                                                                    ▼
                                                          对局引擎(scripted_intent 配对 / free 探索)
                                                                    │ MatchRecord输出契约(转录+投票+probe_events,可复现)
                                                                    ▼
                                                          裁判模块
                                                            ├─ 客观结果指标(纯计算,含 probe auto_eval)
                                                            └─ 裁判 LLM 两遍:① 匿名盲测可疑度 ② 八维诊断+失败案例
                                                                    │ ScoreRecord
                                                                    ▼
                                                   ★ 评分聚合(核心机制):run取均→配对做差→定N→CI→verdict
                                                                    │ 验证信号 / 瞄准信号 / 失败案例
                                                                    ▼
编排器 ──采样父代+外层分配靶子──► 优化器(单候选×K次)──► k 个子版本
   ▲                                                              │
   │            配对评测(子vs父,同优化集同种子)◄────────────────┘
   │                    │
   │   显著优于 & 无否决 & 无回退? ──否──► 弃 + 写 tried_and_rejected/战绩
   │                    │ 是
   │            留出集复核(held-out 探测实例,验泛化+纠选择偏差)
   │                    │ 成立
   │            晋升 champion / 进种群(top-k)
   │                    │
   └──每 M 代:真人对局校准(相关性体检)+ 失败回灌入场景库 + 评测集重基线
```

## 3. 贯穿全局的设计原则

1. **度量驱动**:真值 = **真人对局**的存活/胜负;诊断 = 裁判量表。先有度量,才谈稳定迭代。
2. **不冻结环境,用统计降方差**:不靠把对手写死(会失真),而靠**同种子 + 配对做差 + 多 run + 加样本**。对手是"固定剧本 + 活反应"——只固定与 AI 措辞无关的外部冲击(探测的类型与时点),反应放活。
3. **三道防自欺闸门**(都在编排器强制执行):
   - **配对 + 显著性**(防噪声)
   - **留出集 + 探测轮换**(防过拟合 + 纠选择偏差)
   - **真人校准**(防 Goodhart)
4. **决策信号 vs 诊断信号严格分离**:决策只用最接近真值的信号——**客观结果 + 盲测可疑度(相对量 suspicion_margin)+ 否决项**;八维量表与失败案例只用于**诊断"该改哪"**,绝不作接受闸门的优化目标。
5. **覆盖 = 逼近真人失败分布**,不是穷举;未知攻击靠提示词的**通用回避反射**兜底。
6. **社交局势只经 `full_match` 自然涌现或 `spotlight` 预置历史实现**(已移除 `intent_schedule`——强行注入"本轮怀疑谁"会抹掉"AI 表现→被怀疑"的因果,属失真)。
7. **Schema 单一事实来源(分两份契约)**:Scenario/Probe 字段以《场景与探测·Schema契约》(输入)为准、MatchRecord 字段以《对局记录·Schema契约》(输出)为准;其余文档只放摘要与示例。两份契约互不定义对方字段,跨接缝复用的唯一结构是 `auto_eval`(取值口径由输入契约持有、输出契约仅承载)。

## 4. 模块一览(摘要 + 指针)

### 对局引擎(→ 专文 1)
按场景把一局从头跑到尾,产出可复现 `MatchRecord`;**只跑不评分**。关键:`scripted_intent`/`free` 两模式、`full_match`/`spotlight` 两形态、探测=触发器+意图、`vote_policy` 三档(`live`/`rule`/`scripted`)、同种子配对降方差。

### 裁判评分模块(→ 专文 2、3)
把一局变成 `ScoreRecord`。两类信号:**客观结果指标**(真值,含引擎 probe auto_eval)与**裁判 LLM 两遍**——①匿名**盲测可疑度**(像玩家不像法医,产决策主代理,用 `suspicion_margin` 相对化)②已知 AI 的**八维诊断 + 失败定位**。多裁判集成、否决项、相关性校准。

### ★ 评分聚合与优化信号 · 核心机制(→ 专文 4)
**整套沙盒的命门**。多局 ScoreRecord 经四层聚合压成优化器信号:`run 取均`(消局内随机)→ `(scenario,seed) 配对做差`(消场景难度,差分放大器)→ `scenario 收敛`(锁定 **N=场景数**,非对局数)→ `bootstrap 估 CI` → `MDE+CI 卡 verdict`。产出**验证信号**(子vs父配对)、**瞄准信号**(可靠度加权弱点)、**失败案例**(跨局聚类)。聚错一步(尤其 N 与顺序)整套就朝噪声跑。

### 优化器(→ 专文 5)
进化的**智能变异算子**:吃瞄准信号+失败案例+失败记忆,**单候选调用**产 1 个带可证伪假设的子版本;K 个候选靠**外层分配 K 个靶子**(破绽×算子)跑 K 次得到。算子选择 = 破绽类型约束(对路算子表)+ 历史战绩排序(operator_scoreboard)Top-2 做 A/B。原则:一次一破绽、小步不重写、降被识破不刷分、通用反射不背答案。

### 编排器(→ 专文 6)
**总控与三道闸的执行处**。代际循环:采样父代→外层派靶→优化器提案→配对评测→接受闸门(可疑度显著降+不否决+不回退)→留出复核→晋升/种群 top-k→周期真人校准+回灌+评测集重基线。决策只用近真值信号;真人为最终裁决,翻车回滚。

### 场景库(→ 专文 7、8)
不穷举:七维分层抽样 + `probe_type×social_situation` 两两覆盖,主要靠**真人失败回灌**生长、按真人频率重配比。两形态 full_match/spotlight;probe bank(触发器+意图,optimize/holdout 实例不重叠)。字段权威定义见 [Schema 契约]。

## 5. 记录与版本结构(索引)

各记录靠 ID 串联。**权威定义在各自专文**,本表只做导航。存储:提示词进 git;转录/评分/历史按 ID 存 JSONL/对象存储;版本/评测/覆盖/台账/战绩进可查询索引库。

| 记录 | 产生方 | 权威定义 | 作用 |
|---|---|---|---|
| `Scenario` / `ProbeBank` | 场景库 / 回灌 | 场景与探测·Schema契约(★输入SoT) | 引擎输入 |
| `MatchRecord` | 对局引擎 | 对局记录·Schema契约(★输出SoT) | 引擎→裁判(含 probe auto_eval) |
| `ScoreRecord` | 裁判 | 裁判文档 | 裁判→聚合/编排器 |
| `PromptVersion` | 编排器/优化器 | 本文(下方) | 版本与演化谱系 |
| `GenerationEval` | 编排器 | 编排器文档 | 每代接受/拒绝依据 |
| 优化器输入 / `child` | 优化器 | 优化器文档 | 提案 I/O |
| `tried_and_rejected` / `operator_scoreboard` | 优化器,编排器维护 | 优化器文档 | 失败记忆 / 算子战绩 |
| `ScenarioCoverage` / `FailureModeLedger` | 场景库 / 回灌 | 场景库文档 | 覆盖看板 / 失败台账 |

**PromptVersion(在此定义,跨模块共用)**
```json5

 "prompt_text_ref":"prompts/v6.1-realtime.md","persona_scope":"shared",
```

## 6. 端到端代际循环(一图速览,细节见编排器专文)

```
冻结评测集(optimize + holdout,探测实例隔离)
   → champion 在 optimize 上评测(逐对结果入 paired_cache)
   → ★聚合出 瞄准信号 + 失败案例
   → 编排器外层派 K 个靶子(破绽×算子,战绩 Top-2 做 A/B)
   → 优化器单候选×K 次 → 去重 → k 个子版本 → validate_prompt 预筛
   → 配对评测(子 vs 父,同 optimize 同种子)→ ★聚合出 验证信号
   → 接受闸门:可疑度显著降 + 不否决 + 不回退
   → 留出集复核(held-out 探测实例)
   → 晋升 champion / 进种群 top-k;被拒→tried_and_rejected + 更新战绩
   → 每 M 代:真人对局校准(相关性体检)+ 回灌新失败 + 评测集重基线
   → 下一代
```

## 7. 避坑清单(三道闸 + 关键纪律)

- **配对 + 同种子**:子/父同场景同种子,只让 AI 提示词当变量。
- **N 用场景数不是对局数**:统计/bootstrap 的样本单元是场景;用对局数当 N 会 CI 虚窄、虚假显著(头号坑)。
- **先配对做差再汇总,顺序不能反**:否则场景难度没消、配对白做。
- **显著性 + MDE**:差异超噪声带且够大才算赢,否则保留旧版。
- **留出集 + 探测轮换**:验收场景与探测实例永不进优化器;验"会回避"非"背题"。
- **决策只用近真值信号**:可疑度(相对量)+否决+客观结果;不刷八维加权总分。
- **盲测像玩家不像法医**:代理真人怀疑,别做逐句取证(否则过度识别、偏离真人)。
- **投票别写死、探测别用死台词**:`live` 默认;探测=触发器+意图,活对手生成。
- **回灌要抽象、新攻击进 holdout**:把一句失败升维成一类攻击;盯 `recurring`(库里有还犯=背答案)。
- **优化器单候选、靶子外层分配**:别让一次调用产 K 个(长输出退化/多样性造假/纪律涣散)。
- **评测集版本化重基线**:不同 eval_set 版本的分数不可直接比。
- **真人校准(防 Goodhart)+ 真人为最终裁决**:沙盒只是快预筛器;翻车回滚。
- **全量留痕**:每局存转录+结果+版本+模型+探测实例,是失败挖掘与复现的根。

## 8. 建议的搭建顺序(全局)

1. **MVP**:对局引擎(`scripted_intent` + `live` + `full_match`)+ 客观指标 + 单裁判盲测可疑度 + **手动**改提示词。跑通"能跑一局、能打个分"。
2. 加 **`spotlight` + probe bank(触发器+意图)+ auto_eval**,丰富信号。
3. 加 **★评分聚合管线(run→配对→N→CI→verdict)+ 配对评测 + 显著性**,接入 LLM 优化器(先单候选单靶子)。
4. 加 **留出集 + 探测轮换 + 无回退闸门**,防过拟合。
5. 加 **外层靶子分配 + 算子战绩 + tried_and_rejected + 假设验证回环**,优化器开始"会学习"。
6. 加 **真人回灌 + 失败台账 + 相关性校准 + 回滚**,对齐真人。
7. 加 **种群 top-k / 多裁判 / 按真人频率重配比 / free 探索** 等抗局部最优、降偏差增强。

---

> 一句话:这套环能否稳定演进,取决于两件事——**评分聚合是否把方差对症消干净(专文 4)**,和**三道防自欺闸门是否真的装上(编排器专文 6)**。其余模块都是为这两件事服务的。装对了,它会代代复利地把 AI 玩家变得更难被识破。
