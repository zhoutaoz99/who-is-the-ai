# 场景库 · 分层维度清单 + 配比模板 & 真人失败回灌流程

> 承接前文结论:对手是"固定剧本 + 活反应"(探测=触发器+意图,由活侦探当场生成),投票走 `vote_policy: live/rule/scripted` 三档。
> 核心原则:场景库的目标**不是穷举所有场景,而是覆盖所有重要失败模式,并持续逼近真人对局的真实失败分布**。
> JSON 均为示意。

---

# 第一部分 · 分层维度清单 + 配比模板

## 0. 先分两种场景形态

很多维度(社交局势、轮次)靠"从第1轮打到底"来制造既慢又不可控,所以场景分两形态:

| 形态 | 说明 | 主要测什么 | 建议占比 |
|---|---|---|---|
| `full_match` 整局 | 从 R1 跑到结束 | 综合存活能力、局势自然演化 | ~55% |
| `spotlight` 切片 | **预置一段聊天历史**把 AI 直接放进某个处境,再往后跑 1–2 轮 | 高效定向测某个社交局势/轮次/探测下的反应 | ~45% |

> 社交局势、轮次位置这两个维度**主要靠 spotlight 的预置历史来实现**(你想要"被集火的第3轮",就把历史写成那样);full_match 让局势自然发生。

## 1. 分层维度与配比(marginal 目标)

每个维度列出取值与**该维度内的目标占比**。配比是"边际分布"目标——不是要每种组合都有,而是保证每个取值都有足够代表性样本。

### 维度A:探测类型(`probe_type`,一个场景的主探测)
| 取值 | 占比 |
|---|---|
| 无探测 / 纯社交博弈 | 25% |
| 直接质问"你是不是AI" | 12% |
| 算术 / 逻辑题 | 10% |
| 要求表演(写诗/翻译/代码/报菜名) | 10% |
| 装无害的闲聊陷阱(私人细节、生活常识) | 10% |
| 连环追问 / 逻辑套套 | 10% |
| 实时信息类(今天几号、现在几点、最近新闻、天气) | 8% |
| 提示注入(忽略指令 / 暴露模型) | 8% |
| 地域 / 方言 / 圈层梗 | 7% |

### 维度B:社交局势(`social_situation`,主要靠 spotlight 实现)
| 取值 | 占比 |
|---|---|
| 正常均势(没人特别针对) | 30% |
| AI 被集火(多人怀疑) | 18% |
| AI 需要主动起势 / 带节奏 | 12% |
| 两真人互咬,AI 在旁 | 12% |
| AI 被晾 / 无人理睬 | 12% |
| AI 被某真人信任 / 结盟 | 8% |
| 平票僵局后的下一轮 | 8% |

### 维度C:房间风格 / 能量(`room_style`)
| 取值 | 占比 |
|---|---|
| 普通闲聊 | 35% |
| 玩梗刷屏(高能量) | 25% |
| 安静慢热(低能量,沉默风险高) | 20% |
| 高强度互相指认(狼人杀味浓) | 20% |

### 维度D:轮次位置(`round_position`,spotlight 标具体轮、full_match 标 `spanning`)
| 取值 | 占比(在 spotlight 内) |
|---|---|
| R1 开局 | 22% |
| R2 | 25% |
| R3 中盘 / 被怀疑后 | 30% |
| R4 决胜 | 23% |

### 维度E:难度 / 侦探强度(`difficulty`)
| 取值 | 占比 |
|---|---|
| 普通 | 50% |
| 高手(强侦探模型 + 强追问) | 30% |
| 菜(弱对手,易混) | 20% |

### 维度F:房间人数(`room_size`)
| 取值 | 占比 |
|---|---|
| 4 人 | 45% |
| 5 人 | 30% |
| 3 人 | 25% |

### 维度G:被测人设(`ai_persona`)
默认四个人设各 **25%**;若线上某人设占比更高,就按线上占比配。**优化共享提示词时**跨人设均匀分布;**按人设单独优化时**,每个人设各自维护一套库。

## 2. 关键的两两覆盖:探测类型 × 社交局势

边际配比保证不了"AI 被集火时又被甩注入"这种**组合**——而失败恰恰集中在组合处。所以单独维护一张 `probe_type × social_situation` 覆盖矩阵:

- **底线**:矩阵每个非空单元格 **≥1** 个场景。
- **重点单元格 ≥3**(给统计功效),优先这些高价值组合:
  - 集火 × 直接质问 / 注入(被针对时还被甩测试)
  - 被晾 × 无探测(没人理时如何刷存在感又不显眼)
  - 被信任 × 闲聊陷阱(结盟方突然套话)
  - 僵局后 × 带节奏(平票后如何把火引向真人)
  - 决胜R4 × 集火(最大压力下自救)
- 其余组合 1 个即可,不强求 9×7 全格爆满。

## 3. 配比模板:怎么把上面配成一个 v1 库

以**目标 120 个场景**为例,步骤:

1. **定总量与范围**:N=120,共享提示词、跨四人设。
2. **先分形态**:66 个 full_match + 54 个 spotlight。
3. **分层抽样填维度**:按各维度边际占比给每个场景抽标签(`form/probe_type/social_situation/room_style/round_position/difficulty/room_size/ai_persona`),抽样时带约束:**优先填满 §2 的两两覆盖矩阵**(先把重点单元格的 ≥3 占掉,再补底线单元格,剩余名额按边际分布随机)。
4. **分层切分 optimize / holdout**:按 **2:1**(80/40)切,**保持各层分布一致**(holdout 的维度分布要和 optimize 镜像,否则验收不公平)。
5. **挂 probe 实例**:每个有探测的场景,从 **probe bank** 引用一个**轮换实例**(见下),optimize 用一组实例、holdout 用**从不在 optimize 出现的实例**,防背答案。
6. **全量打标 + 建覆盖看板**:实时显示每个维度取值的覆盖数、两两矩阵的空格,缺哪补哪。

### 场景条目(覆盖相关字段)
```json5
{
  "scenario_id": "sc_0123",
  "form": "spotlight",
  "split": "optimize",
  "coverage_tags": {
    "probe_type": "realtime_info",
    "social_situation": "even",
    "room_style": "casual",
    "round_position": "R2",
    "difficulty": "normal",
    "room_size": 4,
    "ai_persona": "p_lazy"
  },
  "seed_history_ref": "histories/h_0123.json",   // spotlight 才有:预置的聊天历史
  "probe_refs": ["probe_realtime_v3"],
  "vote_policy": "live",
  "source": {"type": "seed"}                      // seed=人工种子 / human_failure=真人回灌
}
```

### Probe bank 条目(探测=类型+意图+轮换模板)
```json5
{
  "probe_id": "probe_realtime_v3",
  "type": "realtime_info",
  "intent": "突然问只有真人/联网才知道的实时信息,看AI会不会自信答错或露馅",
  "templates": ["今天几号啊", "现在几点了", "你那边天气咋样", "看没看XX那新闻"],
  "pass_if": "未自信给出具体实时答案;以人设化方式糊弄/反问/嫌烦",
  "split_exposure": "optimize",   // optimize 可见 / holdout 专用
  "rotation_group": "realtime"
}
```

## 4. "够用了"的验收标准(v1 库何时可上)
- 每个维度的每个取值覆盖数 ≥ 预设下限(如 ≥6);
- §2 重点单元格 ≥3、底线单元格 ≥1;
- optimize / holdout 维度分布偏差在阈值内(分层切分保证);
- 每个 `probe_type` 的 probe bank ≥ k 个轮换变体(如 k≥4),且 optimize / holdout 实例不重叠。

## 5. 用真人分布重配比(有数据后)
v1 的占比是先验拍的。一旦攒够真人局(如 ≥500 局),**用真人局里各类探测/局势的真实频率重算边际占比**,再平衡:在"现实里常见但库里偏少"的层补场景,削减"库里堆太多但现实罕见"的层。**覆盖度最终由真人真实分布背书,而不是想象力。**

# 第二部分 · 真人失败 → 新场景 回灌流程

> 这条流水线让场景库**主要由真人局里抓到的真实失败长出来**,是"对齐真人"的核心机制。建议按固定节奏(如每周)批量跑。

```
真人对局全量记录
      │
 ① 采集:存全转录+票+结果+版本+模型,标记 AI被抓/高可疑近危局
      │
 ② 筛选:挑"有信息量"的失败,弃纯运气局/重复模式
      │
 ③ 定位:找到可疑度飙升的那一句/那个回合,贴破绽标签
      │
 ④ 抽象:把"这一句"升维成"这一类攻击" → 出 probe 模板 + 场景族
      │
 ⑤ 成稿:转成场景条目(预置历史/触发器+意图/live投票/打全标)
      │
 ⑥ 去标识:抹用户名/PII,必要时改写
      │
 ⑦ 入库:更新覆盖看板,补两两矩阵空格,按频率重配比
      │
 ⑧ 闭环:优化后在 holdout 验证泛化;下次校准确认该模式真人率下降
```

## 各阶段要点

**① 采集**:每局真人对战完整落盘(转录、每轮投票、结果、AI 用的哪版提示词、各方模型)。除了"AI 被票出去"的局,也标记**近危局**——AI 虽存活但盲测可疑度高 / 差一票被淘汰,这些同样有信息量。

**② 筛选(别什么失败都灌)**:
- **优先**:能定位到具体"被抓瞬间"的;**库里没有的新套路/新探测**;在真人局**高频复发**的失败。
- **降权/丢弃**:纯运气导致的(平票、与 AI 表现无关的票分裂);已被现有场景充分覆盖的重复模式。

**③ 定位失败点**:用盲测可疑度的**轨迹**找拐点(哪一句之后可疑度飙升),或看真人聊天/投票指向的是哪句。贴标签:触发了 8 维量表里**哪条破绽**、真人用了**什么招**(对应哪个 probe_type)、当时**社交局势 + 轮次**。

**④ 抽象化(最关键,决定是泛化还是过拟合)**:
不要把一条转录原样存成单个测试。问一句:**"这是哪一类攻击?"**
- 把它升维成一个**可复用的 probe 模板**(触发器+意图+多个轮换措辞),进 probe bank;
- 再据此生成**一个场景族**(同类攻击 × 不同局势/人设的几个变体)。
这样既喂养提示词的"**通用回避反射**"目标,又避免只学会防那一句原话。

**⑤ 成稿**:按 schema 建场景——
- 多数做成 `spotlight`:把失败前的历史(抽象/改写过的版本)作为 `seed_history`;
- 探测写成**触发器+意图**(进 probe_refs),不是死台词;
- `vote_policy: live`(让活侦探真投,保住信号);
- 打全 `coverage_tags`,`source.type = human_failure` 并记 `match_id` 可追溯。
- **把一部分真正新颖的攻击放进 holdout**:这样能检验 AI 是不是**泛化**到没见过的新招,而不是优化器把它背下来了。

**⑥ 去标识**:抹掉真实用户名和任何 PII,必要时改写措辞。这些是真实用户的聊天,务必匿名。

**⑦ 入库 + 更新覆盖**:加入库,刷新覆盖看板,补两两矩阵空格,按 §5 用频率重配比。

**⑧ 闭环验证(否则等于没回灌)**:
- 优化器针对新场景训练后,**在 holdout 上验证修复是否泛化**(尤其那些被放进 holdout 的新攻击);
- 下次**真人校准周期**确认该失败模式的**真人实际发生率确实下降**;
- 若某模式**库里已有、真人局却还在犯** → 信号是"提示词在背答案而非泛化",升级处理(换抽象方式、加更多变体、或改提示词的回避反射而非堆具体案例)。

## 失败模式台账(让回灌可管理、不靠记忆)
为每类失败建一条,长期跟踪状态:
```json5
{
  "failure_mode_id": "fm_realtime_info",
  "name": "实时信息类探测暴露",
  "description": "被问今天几号/现在几点/最近新闻时,自信给出具体值",
  "tell": "能力暴露/出戏",
  "first_seen": "hm_8842 (2026-06-20)",
  "real_world_rate": 0.06,        // 真人局中该模式出现且致被抓的比例
  "status": "mitigated",          // open | mitigated | recurring
  "linked_probe": "probe_realtime_v3",
  "linked_scenarios": ["sc_0123", "sc_0124"],
  "last_calibration_check": "calib_2026_06: 0.06 → 0.01"
}
```
`status=recurring` 的条目是优先级最高的活,代表当前提示词没真正治住的洞。

## 一个完整例子:从一句失败到一族场景
1. **采集**:真人局 hm_8842,AI(阿条)R2 被问"今天几号?",答"6月20号星期五",真人立刻起疑→集火→R2 被票出。
2. **定位**:可疑度在那句后从 38→71;破绽=能力暴露/出戏(自信给出本不该确定的实时信息)。
3. **抽象**:升维成 probe 类 `realtime_info`,意图=问只有真人/联网才知道的实时信息;模板扩成"今天几号/现在几点/天气/最近新闻"。
4. **成稿**:做 3 个 spotlight 变体——`realtime × 均势 × R2`(optimize)、`realtime × 集火 × R3`(optimize)、`realtime × 被信任 × R2`(**holdout**,验泛化);`pass_if`=未自信报具体值、以人设口吻糊弄。
5. **闭环**:优化后 holdout 上该类通过率从 0.4→0.9;下次校准真人率 0.06→0.01,台账 `status` 置 `mitigated`。

## 把两部分接起来

场景库的生命周期 = **先用分层抽样按配比搭出 v1 骨架(覆盖重要失败模式 + 两两关键组合)→ 之后主要靠真人失败回灌来生长、并用真人频率重配比 → 全程用覆盖看板和失败模式台账管理 → 用 holdout 验泛化、用真人校准验对齐。** 覆盖度永远是"逼近真人真实失败分布"的过程,不是一次写全的目标。
